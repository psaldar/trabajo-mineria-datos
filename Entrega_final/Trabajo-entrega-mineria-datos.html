Ajuste de hipeparámetros¶

Con el fin de realizar las combinaciones previamente mencionadas se entranan modelos con diferentes hiperparmetros

  • rank = la cantidad de factores latentes en el modelo (5, 10 y 15 como valores seleccionados)
  • maxIter = el nmero mximo de iteraciones (valor predeterminado 10)
  • regParam = el parmetro de regularizacin (0.01, 0.1 y 0.50 como valores seleccionados)

def als_pipeline(df_train,df_val,s,rank, maxiter):
    
    als = ALS(maxIter=maxiter, 
              rank =rank , 
              regParam=s, 
              userCol="customer_id_integer", 
              itemCol="item", 
              ratingCol="star_rating_integer",
              coldStartStrategy="drop")
    
    model = als.fit(df_train)
    predictions = model.transform(df_val)
    predictions_corr = predictions.withColumn("preds_corr",

Modelos cold start¶

Este modelo usa un MinHashLSH primero para hallar los articulos mas similares a un cierto producto basandose en el ttulo de ste. Este producto podria ser un "producto nuevo" sin ningun problema, lo cual lo hace un modelo apto para el cold start (simplemente se necesita tener la metadata de este articulo). Usando LSH, se hallan los "n" productos mas similares para este articulo. Luego de este paso, en una segunda etapa se usan las variables de precio, categora y marca del artculo y de sus "n" artculos ms similares para computar un score de afinidad, y con este score podremos determinar cuales son los "m" artculos ms afines al producto de inters de entre los "n" artculos similares hallados en la etapa 1. Estos "m" artculos que salen del segundo paso seran los artculos que se recomiendan junto con este artculo. As, a las personas que hayan comprado alguno de los "m" productos, se les debe recomendar tambin el producto de inters.

Para validar el rendimiento de este modelo, se extraen algunos artculos del conjunto de datos general y se dejan aparte en un conjunto de validacin. Para predecir los ratings de estos artculos, se toma el promedio de los "m" artculos ms afines segn este procedimiento. ste sera el rating predicho para este artculo (sin importar el usuario) y con este rating se calculara la mtrica (RMSE) del modelo (comparando los ratings predichos contra los reales) para ver su desempeo.

def obtiene_precio(precio):
    precio1 = precio.replace('$','')
    precio2 = precio1.replace(',','')
    precio3 = precio2.split(' ')
    precio4 = 0
    count=0
    for i in precio3:
        if len(i.strip())>1:
            count = count+1
            precio4 = precio4+float(i)
    return precio4/count


### Funcion para calcular la similaridad de Jaccard (para las listas)
def jaccard_similarity(list1, list2):
    s1 = set(list1)
    s2 = set(list2)
    return len(s1.intersection(s2)) / len(s1.union(s2))

(Nota: Tener en cuenta que aqu el RMSE se calcula comparando el rating predicho para cada producto con el rating promedio real de cada producto. Se hace con los promedios de los ratings de los productos en vez de con las reviews originales para facilitar los clculos y los procesamientos. Cmo se puede ver, a pesar de no tener un RMSE tan bueno como los modelos de factorizacin de matrices con factores latentes de la seccin anterior que adems incluan Bias, el RMSE obtenido por este modelo es de igual forma decente, por lo que es una buena alternativa para cuando se tenga que dar recomendaciones en el mbito del problema de cold start (cuando hay productos nuevos, el modelo de la seccin anterior no estara diseado para sugerir recomendaciones para esos productos nuevos, pero este modelo si podra usarse en esos casos).
